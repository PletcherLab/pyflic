# pyflic plot inventory

This document lists **all plots producible by the current pyflic codebase** for FLIC data, and the **experiment types** they apply to.

Experiment types (from `global.experiment_type` in `flic_config.yaml`):

- `single_well` (class `SingleWellExperiment`, `chamber_size: 1`)
- `two_well` (class `TwoWellExperiment`, `chamber_size: 2`)
- `hedonic` (class `HedonicFeedingExperiment`, `chamber_size: 2`)
- `progressive_ratio` (class `ProgressiveRatioExperiment`, `chamber_size: 2`)

Output directories:

- `project_dir/<config_stem>_results/qc[_start_end]/...` is written by `Experiment.write_qc_reports()`.
- `project_dir/<config_stem>_results/analysis[_start_end]/...` is written by analysis methods (and the Analysis Hub GUI).
- Chamber exclusions from `remove_chambers.csv` (selected group) are applied at load time, so all treatment-level plots reflect the filtered design.

---

## QC plots (saved to `.../qc/`; all experiment types)

Generated by `Experiment.write_qc_reports()` (per DFM).

- **Raw signal (all wells)**:
  - **API**: `DFM.plot_raw()`
  - **Saved as**: `qc*/raw_signal/DFM{dfm_id}_raw.png`
  - **Experiment types**: `single_well`, `two_well`, `hedonic`, `progressive_ratio`

- **Baselined signal (all wells; thresholds overlaid)**:
  - **API**: `DFM.plot_baselined(include_thresholds=True)`
  - **Saved as**: `qc*/baselined/DFM{dfm_id}_baselined.png`
  - **Experiment types**: `single_well`, `two_well`, `hedonic`, `progressive_ratio`

- **Cumulative licks (all wells/chambers)**:
  - **API**: `DFM.plot_cumulative_licks(transform_licks=True)`
  - **Saved as**: `qc*/cumulative_licks/DFM{dfm_id}_cumulative_licks.png`
  - **Experiment types**: `single_well`, `two_well`, `hedonic`, `progressive_ratio`

## Experiment-level summary plots (saved to `.../analysis/`)

### Feeding summary (all experiment types; metric set depends on type)

- **What it is**: faceted “box + jitter” panels, grouped by treatment (or by factor-combination when `experimental_design_factors` is set).
- **API (returns plotnine `ggplot`)**: `Experiment.plot_feeding_summary(...)`
- **API (saves to disk)**: `Experiment.write_feeding_summary_plot(format="png"|"pdf", ...)`
- **Saved as**: `analysis*/feeding_summary.{png|pdf}`
- **Experiment types**:
  - `single_well` (from `SingleWellExperiment._feeding_plot_metrics()`):
    - **Metrics/columns**:
      - `Licks`
      - `Events`
      - `MeanDuration`
      - `MedDuration`
      - `MeanTimeBtw`
      - `MedTimeBtw`
      - `MeanInt`
      - `MedianInt`
  - `two_well` / `hedonic` / `progressive_ratio` (from `TwoWellExperiment._feeding_plot_metrics()`):
    - **Metrics/columns**:
      - `PI`
      - `EventPI`
      - `LicksA`, `LicksB`
      - `EventsA`, `EventsB`
      - `MeanDurationA`, `MeanDurationB`
      - `MedDurationA`, `MedDurationB`
      - `MeanTimeBtwA`, `MeanTimeBtwB`
      - `MedTimeBtwA`, `MedTimeBtwB`
      - `MeanIntA`, `MeanIntB`
      - `MedianIntA`, `MedianIntB`
    - **Note**: the display labels in the facets use `well_names` (`A`/`B`) when provided in `flic_config.yaml`.

### Binned licks by treatment (mean ± SEM timecourse)

- **What it is**: per-treatment mean line with optional SEM band across time bins (minutes on x-axis).
- **API (returns plotnine `ggplot`)**: `Experiment.plot_binned_licks_by_treatment(...)`
  - (This is a compatibility wrapper over `Experiment.plot_binned_metric_by_treatment(metric="Licks", ...)`.)
- **Saved as (Analysis Hub GUI)**: `analysis*/binned_Licks.png`
- **Experiment types**: `single_well`, `two_well`, `hedonic`, `progressive_ratio`

### Binned metric(s) by treatment (mean ± SEM timecourse)

- **Single metric**:
  - **API**: `Experiment.plot_binned_metric_by_treatment(metric=..., ...)` → plotnine `ggplot`
  - **Valid `metric` values**:
    - **Any column present** in the binned feeding-summary rows (e.g. `PI`, `EventPI`, `LicksA`, `MedDurationB`, etc.).
    - Or one of these **base metric names** (resolved automatically depending on chamber size):
      - `Licks`
      - `Events`
      - `MeanDuration`
      - `MedDuration`
      - `MeanTimeBtw`
      - `MedTimeBtw`
      - `MeanInt`
      - `MedianInt`
  - **Two-well combining (`two_well_mode`)** when using base names and A/B columns exist:
    - `total`: A+B (recommended for `Licks`, `Events`)
    - `mean_ab`: mean(A,B) (recommended for durations/intervals)
    - `A`: Well A only
    - `B`: Well B only
  - **Saved as**: not auto-saved by the core pipeline (call `p.save("out.png", dpi=200)` yourself)
  - **Experiment types**: `single_well`, `two_well`, `hedonic`, `progressive_ratio`

- **Multiple metrics (facet grid)**:
  - **API**: `Experiment.plot_binned_metrics_by_treatment(metrics=(...), ...)` → plotnine `ggplot`
  - **Valid `metrics` entries**: same options as `metric` above (existing columns or base names).
  - **Saved as**: not auto-saved by the core pipeline (call `p.save("out.png", dpi=200)` yourself)
  - **Experiment types**: `single_well`, `two_well`, `hedonic`, `progressive_ratio`

### Generic jitter+mean+SE helper (plotnine)

- **What it is**: reusable “jitter + mean marker + mean±SEM errorbar” plot, with optional faceting.
- **API**: `Experiment.plot_jitter_summary(df, x_col=..., y_col=..., facet_col=..., ...)` → plotnine `ggplot`
- **Saved as**: not auto-saved (use `p.save("out.png", dpi=150)` yourself)
- **Experiment types**: `single_well`, `two_well`, `hedonic`, `progressive_ratio` (depends only on the columns in `df`)

## DFM-level exploratory plots (not auto-saved by default)

These return matplotlib `Figure` objects (save with `fig.savefig(...)`).

- **Raw signal (single well)**:
  - **API**: `DFM.plot_raw_well(well, ...)`
  - **Experiment types**: all

- **Baselined signal (single well; optional thresholds)**:
  - **API**: `DFM.plot_baselined_well(well, include_thresholds=..., ...)`
  - **Experiment types**: all

- **Binned feeding (per chamber, within one DFM)**:
  - **API**: `DFM.plot_binned_licks(binsize_min=..., ...)`
  - **Experiment types**: all

- **Cumulative licks (within one DFM)**:
  - **API**: `DFM.plot_cumulative_licks(single_plot=..., transform_licks=...)`
  - **Experiment types**: all

- **Cumulative licks (single chamber; lights-on markers)**:
  - **API**: `Experiment.plot_cumulative_licks_chamber(dfm_id=..., chamber=..., ...)`
    - (Wraps `DFM.plot_cumulative_licks_chamber(...)` and adds treatment name when available.)
  - **Experiment types**: all

## Two-well / choice-specific plots

### Well A vs Well B duration jitter plot (faceted)

- **What it is**: compares WellA vs WellB for a chosen metric (default `MedDuration`), faceted by treatment.
- **API (returns plotnine `ggplot`)**: `TwoWellExperiment.facet_plot_well_durations(metric="MedDuration", ...)`
- **Valid `metric` values**:
  - Any **base metric name** such that both `{metric}A` and `{metric}B` exist in `exp.feeding_summary()` (common examples):
    - `MeanDuration` → uses `MeanDurationA`, `MeanDurationB`
    - `MedDuration` → uses `MedDurationA`, `MedDurationB`
    - `MeanTimeBtw` → uses `MeanTimeBtwA`, `MeanTimeBtwB`
    - `MedTimeBtw` → uses `MedTimeBtwA`, `MedTimeBtwB`
    - `MeanInt` → uses `MeanIntA`, `MeanIntB`
    - `MedianInt` → uses `MedianIntA`, `MedianIntB`
- **Saved as (Analysis Hub GUI)**: `analysis*/well_comparison_{metric}.png`
- **Experiment types**: `two_well`, `hedonic`, `progressive_ratio`

### Cumulative preference-index plots (two-well only; per DFM)

- **Cumulative PI from licks**:
  - **API**: `DFM.plot_cumulative_pi(range_minutes=..., single_plot=...)`
  - **Experiment types**: `two_well`, `hedonic`, `progressive_ratio`

- **Cumulative PI from events**:
  - **API**: `DFM.plot_cumulative_event_pi(events_limit=..., by_bout=..., single_plot=..., ...)`
  - **Experiment types**: `two_well`, `hedonic`, `progressive_ratio`

## Hedonic-specific plots

### Hedonic feeding plot (WellA vs WellB median durations; faceted)

- **What it is**: a well-comparison jitter+summary plot (WellA vs WellB), typically with point size proportional to event counts.
- **API (returns plotnine `ggplot`)**: `HedonicFeedingExperiment.hedonic_feeding_plot(...)`
- **Metric(s)**:
  - Fixed to **median bout duration**: `MedDurationA` vs `MedDurationB` (plotted as `MedDuration` after melting).
  - Point size uses `EventsA` / `EventsB` when available (as weights/visual emphasis).
- **Saved as (default when `save=True`)**: `analysis*/hedonic_feeding_plot.{format}` (default `png`; in the hub this is written by calling the same API)
- **Experiment types**: `hedonic`

---

## Progressive-ratio-specific plots

### Breaking-point plots (per DFM; faceted by well)

- **API (returns plotnine `ggplot`)**: `ProgressiveRatioExperiment.plot_breaking_point_dfm_gg(dfm, configuration, ...)`
- **Metric(s)**:
  - Plots `DeltaLicks` vs `Minutes` (one facet per well `W1..W12`).
- **Saved as (Analysis Hub GUI / scripts)**: `analysis*/breaking_point_dfm{dfm_id}_config{configuration}.png`
- **Experiment types**: `progressive_ratio`

## Notebook / custom-output examples (same underlying plot functions)

The repository notebooks sometimes save plots under custom names/locations (e.g., a “first 6 hours” subset). These are not additional plot types; they are alternate filenames produced by the same APIs above (e.g. `write_feeding_summary_plot()` and `plot_binned_licks_by_treatment()`).
